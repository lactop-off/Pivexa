## 1. 背景・目的

データをアップロードし、必要な設定と「実行」ボタンの操作だけで統計解析を行えるツールを開発する。専門知識がない利用者でも、回帰分析をはじめとする統計解析を実施し、結果（P値・係数・説明変数など）を確認・理解できる状態を目指す。

提供形態は段階的に拡大する。

- **第1段階（MVP）**：オンプレミス単独版を**フリー**で提供（組織内に Docker Compose で導入）
- **後フェーズ**：クラウド版を**サブスクリプション**で提供（RedMine と同様のデュアル提供モデル）

オンプレ単独版でも非専門家が安心して使えるよう、解釈サポート・推奨値提示などのユーザビリティを重視する。

## 2. ツール概要

汎用的な統計解析ツール。利用者は以下の流れで分析を行う。

```
データをアップロード → 前処理（任意） → 分析手法と変数を設定 → 実行 → 結果表示 → 解釈・出力
```

各分析手法は「設定 → 実行 → 結果 → 解釈」という共通の流れを持つ（詳細は「7. 共通設計方針」）。

## 3. 想定ユーザー

- **提供形態（MVP）**：顧客組織のオンプレミス環境に導入。組織内の複数ユーザーが利用。
- **権限**：ロールは**管理者レベルのみ**（MVP では細かいロール分割はしない）。
- **利用者の前提知識**：統計解析の専門知識は持たない可能性が高い。専門用語の補助や結果の自動解釈が重要。

## 5. 機能要件

### 5.1 データ取り込み

- **CSV・Excel** ファイルのアップロード（対応形式はこの2種）
- 文字コード・区切り文字の自動判定
- ファイルサイズ上限の設定（具体値は基本設計で確定。初期値の目安は後述）
- アップロード時のバリデーション（壊れたファイル・空ファイル・列数の不整合などを検出して通知）
- （後フェーズ候補）複数ファイルの結合、外部データ連携

### 5.2 データ確認・前処理

- 自動プロファイリング：列ごとの型（数値・日付・カテゴリ）、欠損数、最大・最小・平均、重複などのサマリー表示
- 欠損値の処理（補完／削除）
- 型変換、不要列の除外
- カテゴリ項目のダミー変数化（分析時に自動処理）
- **推奨値の自動提示**（ルールベース）。利用者は専門知識がない前提のため、推奨を提示しつつ最終選択はユーザーに委ねる。

#### 前処理 推奨ロジック（ルールベース）

| 状況 | 推奨アクション |
|---|---|
| 数値列に欠損 | 中央値で補完 |
| カテゴリ列に欠損 | 最頻値で補完、または「欠損」カテゴリ化 |
| 欠損率が高い列（目安：50%超） | 列削除を推奨 |
| 高カーディナリティな列（ID的な列） | 説明変数から除外を推奨 |
| 数値に見えるが文字列の列 | 数値型への変換を推奨 |

### 5.3 分析（統計解析）

すべての手法に共通する画面構成：

**(1) 設定画面（実行前）**
- 目的変数の選択（手法により1つ）
- 説明変数の選択（複数）
- 分析手法の選択
- カテゴリ変数の自動ダミー変数化
- （上級）交互作用項・変数変換 ※後フェーズ

**(2) 結果画面（実行後）**
- 説明変数ごとの係数・標準誤差・t値・**P値**
- 有意性の見える化（記号や色分けで「効いている変数」を強調）
- モデル全体の指標（例：R²・調整済みR²・F検定）
- 診断情報（例：多重共線性 VIF など、結果の信頼性チェック）

**(3) 解釈サポート（非専門家向け・差別化ポイント）**
- 生成方式：**テンプレート方式**（指標の閾値に応じた定型文を埋め込む。AI生成は採用しない）
- P値や係数の意味を日本語の文章で自動解説
- 「この変数は影響が大きい／統計的に有意」などの自動ハイライト
- 注意喚起（相関と因果の違い、サンプル数が少ない場合の警告など）

#### 対応する分析手法と表示指標（標準セット＝MVP）

| 分類 | 手法 | 主な用途 | 表示指標 |
|---|---|---|---|
| 記述統計 | 平均・分散・度数分布・ヒストグラム等 | 全体像の把握 | 件数・平均・中央値・標準偏差・最小/最大・四分位・欠損数＋ヒストグラム |
| 関係性 | 相関分析 | 数値×数値の関係 | 相関係数・p値・散布図＋相関行列ヒートマップ |
| 関係性 | クロス集計＋カイ二乗検定 | カテゴリ×カテゴリの関係 | 度数・期待度数・カイ二乗値・p値・Cramér's V |
| 仮説検定 | t検定 | 2グループの平均の差 | 各群平均・差・t値・p値・95%信頼区間・効果量(Cohen's d) |
| 仮説検定 | 分散分析（ANOVA） | 3グループ以上の平均の差 | F値・p値・群間/群内平方和・多重比較(Tukey) |
| 回帰分析 | 重回帰分析 | 結果が数値の要因分析・予測 | 係数・標準誤差・t値・p値・95%CI／R²・調整済R²・F値・VIF |
| 回帰分析 | ロジスティック回帰 | 結果がYes/Noの要因分析・予測 | 係数・オッズ比・p値／疑似R²・的中率・混同行列・AUC |

#### 上級セット（後フェーズ）

| 分類 | 手法 | 主な用途 |
|---|---|---|
| 多変量解析 | 主成分分析（PCA） | 多数の変数を少数に要約 |
| 多変量解析 | 因子分析 | アンケート等の背後にある要因の抽出 |
| 多変量解析 | クラスター分析 | 顧客セグメンテーション等のグループ化 |
| 時系列分析 | トレンド・季節性・移動平均・予測 | 売上推移などの時間変化の分析 |

### 5.4 結果の出力・共有

- レポートの自動生成
- **PDF／画像エクスポート**（HTML テンプレート → PDF 変換）
- 共有リンク／ダッシュボード（後フェーズ候補）
- 定期実行（後フェーズ候補）

## 6. 非機能要件

### 6.1 セキュリティ・データ保護（おすすめ水準）
- 通信時（HTTPS）・保管時の暗号化
- 認証・パスワードのハッシュ保管
- アップロードファイル・入力値のバリデーション（不正ファイル・インジェクション対策）
- 監査ログ
- アップロードされたデータの取り扱いポリシーの明文化
- ※ テナント分離・データ削除依頼の運用フロー（C-4／C-6）は、**オンプレ単独構成のため当面保留**。クラウド化フェーズで再設計する。

### 6.2 アカウント・権限管理
- ログイン認証
- ユーザー管理
- ロールは**管理者レベルのみ**（MVP）

### 6.3 使いやすさ
- 非専門家を前提としたUI
- ガイド表示・推奨値の提示
- わかりやすいエラーメッセージ
- UI 設計は Claude を用いて行う

### 6.4 性能・スケーラビリティ
- 大きいファイル・同時実行への対応
- 重い処理の**非同期化**（実行後に完了を通知する方式）。ジョブキュー（Celery + Redis）を採用。小規模時は同期実行フォールバックを許容。

### 6.5 運用・課金
- 処理ログ・監視
- 料金プラン：MVP は**フリー**（オンプレ）。クラウド版の**サブスク**設計は後フェーズ。

## 7. 共通設計方針

すべての分析手法は **「設定 → 実行 → 結果 → 解釈」** という共通の流れを持つ。手法ごとに中身（計算内容・表示項目）は異なるが、画面の流れは共通化できる。

→ **最初にこの共通フレームを1つ作り込み、以降は手法を差し替えて追加していく**構造で設計・実装する。要件・設計も「共通の分析フレーム＋個別手法」という形で記述する。

## 付録：用語集

| 用語 | 説明 |
|---|---|
| 目的変数 | 知りたい結果。例：売上、来店数、購入有無 |
| 説明変数 | 結果に影響していそうな要因。例：広告費、気温、店舗面積 |
| 係数 | 説明変数が1増えたときに結果がどれだけ動くかを表す値 |
| P値 | その影響が偶然ではなさそうかを示す指標。一般に0.05未満で「統計的に有意」と判断 |
| R²（決定係数） | モデルが結果をどれくらい説明できているかの指標（0〜1、高いほど良い） |
| 重回帰分析 | 複数の説明変数から、数値の結果を分析・予測する手法 |
| ロジスティック回帰 | 複数の説明変数から、Yes/Noの結果を分析・予測する手法 |
| 多重共線性 | 説明変数どうしが似すぎていて、結果の解釈が不安定になる状態 |
| オッズ比 | ロジスティック回帰で、説明変数が1増えたときに「起こりやすさ」が何倍になるかを表す値 |
| VIF | 多重共線性の強さを示す指標。一般に10以上で注意 |
| テナント分離 | クラウドで複数顧客のデータを互いに見えないよう分離すること |
